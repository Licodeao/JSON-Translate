# JSON-Translate

进行国际化业务时，方便提升手动翻译效率

## 背景

在进行国际化业务时，可能遇到在提取中文后，需要手动进行翻译的情况。

对于庞大的老项目而言，中文字符可想而知，甚至于整个部门的所有项目都需要进行国际化

如果在所有项目都进行提取中文后再来手动进行翻译，工作量可想而知，且效率极其低下

于是，在手动翻译这一环节使用编码进行效率提升

定好需求后，选择了 Google translate api，但发现好像最近 api 的某个字段改了，导致接口使用不了

故而想到了离线翻译，但离线翻译针对于某些业务场景名词可能翻译的不够准确

所以，还是选择联网翻译，最终选择了使用百度翻译，其免费额度相对来说已经够用...

## 使用场景

针对于以下场景进行使用：

```json
{
  "你好": "",
  "苹果": ""
}
```

将 key 的翻译结果作为其 value

## 使用步骤

- npm install 安装依赖
- 根据百度翻译官网步骤，获取 appid、salt，进而得到密钥进行翻译请求
- 将package.json中的module设置为commonjs
  - 或将export关键字删除

- 在文件中引入 translateText 方法，并传入相应参数即可

## Tips

- 翻译结果并不是百分百正确
- 由于百度翻译限制不能同时发出过多的请求，所以使用了缓存减少了请求的数量
